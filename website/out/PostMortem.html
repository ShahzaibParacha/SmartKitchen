Our Development Experience

The Good

We as a team had a pleasure working on the app together. We had a good amount of coherence and communication to make it through together. We were able to follow most, if not all, Agile methodologies and principles and engineer the app development process around them. We managed to get 83% Test Coverage overall.

The Bad

HSQLDB. Malfunctioning Integration Tests. Android Studio and it's shortcomings for days. Designing UI's for different screen sizes.

In Hindsight...

Pair Programming was one of the better decisions we made. We set time aside to work on the app together. This helped us manage problems as soon as they showed up. Five heads thinking together is better than one. This mainly helped us when we were implementing the Database, and Testing because we had delegated tasks and the pair programming aspect allowed us to communicate to manage time and efforts well.

Implementing HSQLDB felt like interacting with technology that will turn into legacy software soon (hopefully..). It would be better to research more database options and incorporate something that is well-documented and will likely stay alive longer. This would have also helped us in testing cause we had issues when copying over Database scripts.

Knowing what we know now, if we were to start over there are a few things that we would do differently. We would add more features and gain more horizontal functionality in our app. This would allow us to iterate better on those features instead of implementing them at once without much room to grow and correct organically. We would also vote on features we see fit, and only include them if there is majority support, so that less valuable features are not taking priority over the more important features.
